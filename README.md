# Тёплый город

Учебный веб-проект агентства недвижимости с регистрацией, авторизацией, каталогом объектов, личным кабинетом агента и базовыми CRM-интеграциями.

## О проекте

`Тёплый город` — это веб-приложение для демонстрации цифрового сервиса агентства недвижимости.  
Проект позволяет:

- регистрировать и авторизовывать пользователей;
- просматривать публичный каталог объектов;
- создавать, редактировать и удалять объявления;
- отправлять заявки на просмотр;
- отправлять сообщения через форму обратной связи;
- автоматически создавать записи в календаре, CRM и журнале SMS/Push-уведомлений.

## Основной функционал

- регистрация и вход в аккаунт;
- роли пользователей: `client`, `agent`, `admin`;
- публичный каталог недвижимости;
- фильтрация объектов по типу, комнатам и цене;
- личный кабинет агента;
- CRUD-операции с объявлениями без перезагрузки страницы;
- загрузка фотографий для объявлений;
- экспорт объявлений в JSON;
- форма обратной связи;
- заявки на просмотр;
- встроенные локальные интеграции:
  - `CRM`;
  - `Календарь`;
  - `SMS / Push`.

## Что хранится в проекте

Основные данные лежат в файле [`db.json`](./db.json):

- `users` — пользователи и роли;
- `properties` — объявления недвижимости;
- `messages` — сообщения с формы обратной связи;
- `viewings` — заявки на просмотр;
- `notifications` — SMS и push-уведомления;
- `calendarEvents` — календарные события;
- `crmLeads` — CRM-лиды.

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/<your-username>/tepliy-gorod.git
cd tepliy-gorod
```

### 2. Запуск проекта

```bash
npm start
```

или

```bash
node server.js
```

### 3. Открытие в браузере

По умолчанию сайт запускается на:

```text
http://localhost:3000
```

### 4. Смена порта при необходимости

PowerShell:

```powershell
$env:PORT=8080
node server.js
```

## API

### Авторизация

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/auth/me`

### Объявления

- `GET /api/properties/public`
- `GET /api/properties`
- `POST /api/properties`
- `PUT /api/properties/:id`
- `DELETE /api/properties/:id`

### Заявки и обратная связь

- `POST /api/feedback`
- `POST /api/viewings`
- `GET /api/viewings`

### Интеграции

- `GET /api/integrations/summary`

## Ограничения учебной версии

- вместо полноценной базы данных используется `db.json`;
- сессии хранятся в памяти процесса Node.js;
- SMS, Push, CRM и календарь реализованы как локальные встроенные интеграции без внешних сервисов.
